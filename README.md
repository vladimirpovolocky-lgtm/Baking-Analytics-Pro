## 📌 Огляд проєкту

## 🛠 Технічна реалізація

### 1. SQL: Трансформація та Логіка (`sql.txt`)
Я розробив архітектуру запиту, яка автоматизує розрахунок ключових бізнес-метрик:
* **Loss-Adjusted Profit:** Розрахунок чистого прибутку з урахуванням повної вартості списаного товару (COGS включає як продані, так і втрачені одиниці).
* **Window Functions (`SUM() OVER`):** Використані для розрахунку загальної частки кожного продукту в прибутку мережі (база для ABC-аналізу).
* **Data Integrity:** Використання `COALESCE` для обробки порожніх значень списань та `CASE WHEN` для запобігання діленню на нуль при розрахунку **Waste Ratio**.

### 2. Візуалізація: UI/UX в Looker Studio
Дашборд розроблений у професійному Dark Mode з акцентом на швидке прийняття рішень:
* **ABC-аналіз (Bubble Chart):** Візуалізація «Маржа vs Продажі» для ідентифікації товарів-локомотивів.
* **Тренд списань:** Автоматичне підсвічування аномальних втрат (якщо показник > 8%) за допомогою умовного форматування.
* **Store Benchmarking:** Теплова карта для порівняння ефективності 12 локацій за прибутком та дисципліною замовлень.

## 💡 Бізнес-аналітика (Key Insights)

Як аналітик, я виділив три стратегічні напрямки для менеджменту:

* **Оптимізація об'ємних драйверів:** Хліб **«СІРИЙ»** та **«БІЛИЙ»** генерують основний трафік, але мають маржу нижче 20%. Рекомендовано провести переговори з постачальниками для зниження собівартості.
* **Управління піковими втратами:** У святковий період (01.01 - 03.01) списання зросли до **15.08%**. Необхідно впровадити окремий план випікання для святкових днів.
* **Категорійний аналіз:** Хліб **«З ТРАВАМИ»** та **«ЦІЛЬНОЗЕРНОВИЙ»** потребують точнішого прогнозування, оскільки через низькі обсяги продажів будь-яке зайве списання робить позицію збитковою.
